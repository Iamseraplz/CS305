# Artemis Financial Security Enhancement Project

## CS 305 Software Security Portfolio Reflection

Artemis Financial is a company that develops individualized financial plans for its clients, including savings, retirement, investment, and insurance services. The company wanted its web application updated to improve security and better protect customer information. To address these concerns, I refactored the application by implementing SHA-256 checksum verification, generating a self-signed SSL certificate, and enabling HTTPS communication. One thing I did well during this project was identifying security weaknesses and applying appropriate solutions to address them. Secure coding is important because it helps protect sensitive data, reduces the risk of security breaches, and improves the reliability of an application. Software security adds value to a company by helping maintain customer trust and protecting important business and customer information. The most challenging part of the vulnerability assessment was configuring the SSL certificate and troubleshooting dependency-check issues, but it was also one of the most helpful parts because it gave me hands-on experience implementing security features in a real application.

I increased the application's security by adding checksum verification, encrypting communications through HTTPS, and reviewing the code to verify that the security changes functioned correctly. After refactoring the application, I tested the `/hash` endpoint and verified that the application generated the correct SHA-256 checksum while running securely over HTTPS. I also used OWASP Dependency-Check (when it worked for me) to assess potential vulnerabilities in project dependencies, although the scan encountered NVD API rate-limiting issues that prevented a complete report from being generated. Resources and tools that were helpful during this project included Java Keytool, Spring Boot, OWASP Dependency-Check, SSL certificates, and secure code review practices. If I were showing this project to a future employer, I would highlight my ability to refactor an existing application, implement secure communications, apply cryptographic techniques, troubleshoot technical issues, and verify that security improvements did not negatively impact the functionality of the software.
